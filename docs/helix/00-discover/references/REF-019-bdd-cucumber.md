# BDD / Cucumber Lineage

Behavior-Driven Development established that human-readable, scenario-style specifications can double as executable tests. Direct precursor to DDx's acceptance-criteria-as-contract pattern: each AC item is the deterministic shape a scenario takes in the bead model.
