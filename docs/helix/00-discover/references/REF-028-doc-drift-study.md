# Documentation Drift (Springer, 2023)

Empirical evidence that documentation rots quickly when it is decoupled from the build/test loop. Motivates DDx's stance that governing artifacts must be executable or referenced from runtime checks (acceptance criteria, contracts, evidence trails) — not prose to be read once and forgotten.
