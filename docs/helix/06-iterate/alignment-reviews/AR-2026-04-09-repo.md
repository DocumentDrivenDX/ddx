## Intent Summary

- **Vision**: DDx is the local-first platform for document-driven, agent-assisted development. Documents remain the system of record for intent; DDx provides the shared runtime and storage primitives around them.
- **Requirements**: The PRD requires stable CLI/server/site outputs, reusable beads and agent execution infrastructure, document graph integrity, process metrics, and a stable plugin API.
- **Features / Stories**: The live alignment pressure is concentrated in FEAT-003, FEAT-004, FEAT-006, FEAT-008, FEAT-016, and FEAT-018.
- **Architecture / ADRs**: The architecture document remains authoritative but its `depends_on` frontmatter does not yet cover all active features with architectural impact. ADR-002 still conflicts with `concerns.md` on frontend lint/test tooling.
- **Technical Design**: FEAT-006 has current SD/TD coverage (`SD-015`, `TD-006`). FEAT-016 and FEAT-018 still have feature specs but no solution design authority.
- **Test Plans**: Frontend and microsite Playwright suites exist, but the declared CI/deploy quality gates are only partially wired.
- **Implementation Plans**: Three active design plans still lack `ddx:` frontmatter, so they remain outside the doc graph and staleness machinery.
