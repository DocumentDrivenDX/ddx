Work moves when the load, lever, and fulcrum are visible. DDx operating principle

The gap

AI agents can write code. They cannot hold context across a codebase, a sprint, or a team.

DDx closes that gap. Specs encode intent. Beads carry acceptance criteria forward. Agents execute against a defined contract — and leave evidence when they are done.

How it works

01

Track work as beads

Create beads with acceptance criteria and dependency graphs. The queue tells agents what to do next.

02

Execute in isolation

Each bead runs in its own worktree. Results are merged — or preserved — with a full audit trail.

03

Review and close

A review pass checks every acceptance criterion. Passing beads close. Failing beads re-queue with findings.

Principles

01 / lever

Platform, Not Methodology

DDx provides primitives. Workflow opinions belong in plugins.

02 / load

Project-Local by Default

Init and install write under the project, keeping machine state out of the product.

03 / fulcrum

Beads Are Work

Every task is a bead with acceptance criteria, dependencies, and audit history.

04 / lever

Cost-Tiered Throughput

Optimize closed work per dollar, with stronger models reserved for harder calls.

05 / load

Design for Simplicity

Start small, name the moving parts, and remove complexity that earns no keep.

06 / fulcrum

Validate Your Work

Good decisions expose the evidence needed to tell whether they are working.
