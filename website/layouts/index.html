The gap

AI agents can write code. They cannot hold context across a codebase, a sprint, or a team.

DDx closes that gap. Specs encode intent. Beads carry acceptance criteria forward. Agents execute against a defined contract — and leave evidence when they are done.

How it works

01

Track work as beads

Create beads with acceptance criteria and dependency graphs. The queue tells agents what to do next.

02

Execute in isolation

Each bead runs in its own worktree. Results are merged — or preserved — with a full audit trail.

03

Review and close

A review pass checks every acceptance criterion. Passing beads close. Failing beads re-queue with findings.

Principles

01

Prompts scatter across machines

Every developer keeps their own folder of half-edited instructions. None of them ship with the project.

02

Specs drift from reality

The spec said one thing, the code did another, and nobody updated the doc the agent reads tomorrow.

03

Work has no shared queue

Tickets live in one tool, AI tasks in another, and neither knows which agent is allowed to pick them up.

04

Agent runs leave no evidence

You cannot tell which prompt, which model, or which inputs produced the diff that just landed.

05

Strong models do cheap work

Routine edits burn frontier tokens because there is no ladder for which model handles which class of task.

06

Improvements stay trapped

A pattern that works in one repo never reaches the next, because nothing shares it back to a library.
