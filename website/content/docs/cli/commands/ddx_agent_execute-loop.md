## ddx agent execute-loop

Removed compatibility surface.

Use `ddx work` for queue draining. The public execution stack is:

1. `ddx run` invokes an agent once.
2. `ddx try` wraps one bead attempt around `ddx run`.
3. `ddx work` wraps queue selection, retry ownership, and tracking around
   `ddx try`.

DDx owns bead success classification and retry decisions. The agent owns model
and provider routing inside a requested power range.
