## ddx bead ready

List beads ready for work

### Synopsis

List beads in the lifecycle-derived ready bucket, sorted by priority.

Ready work has status=open, closed dependencies, and no active cooldown,
external blocker, supersession, ineligible marker, or epic-container exclusion.
Use --execution to include stale in_progress claims that ddx work can reclaim.

```
ddx bead ready [flags]
```

### Options

```
      --execution       Filter to the execution-ready subset (what ddx work picks from): open, deps-closed, not an epic, execution-eligible, not superseded, not on retry cooldown
  -h, --help            help for ready
      --include-human   Deprecated no-op; operator-attention work is status=proposed and excluded from ready output
      --json            Output as JSON
```
