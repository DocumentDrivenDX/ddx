    <title>worker: workerprobe emits periodic worker.heartbeat events with fail-open when no server</title>
    <description>
PROBLEM&#xA;The workerprobe mirrors loop events via `TeeJSONL`, so during a long-running attempt with no intermediate loop event, the server&#39;s `last_event_at` stays at claim time. The server cannot distinguish an actively-working worker from a silent/stale one. The new periodic liveness mechanism must also fail open: when no server address is configured or the server refuses connections, work must continue without errors, status churn, new failure states, or cooldowns.&#xA;&#xA;ROOT CAUSE WITH FILE:LINE&#xA;- `cli/internal/agent/work/heartbeat.go:13-35` ticks but emits no event to the probe.&#xA;- `cli/internal/agent/workerprobe/probe.go:209-228` `RecordEvent` already best-effort posts to `/api/workers/&lt;id&gt;/event` when connected and buffers when not — but nobody calls it for heartbeat-cadence events.&#xA;- `cli/internal/server/worker_ingest.go:110-120` updates `last_event_at` only when `/api/workers/&lt;id&gt;/event` receives an event.&#xA;- `cli/cmd/execute_loop_shared.go:174-186` connects the probe but only `TeeJSONL`-mirrored loop events flow through it.&#xA;&#xA;PROPOSED FIX&#xA;Wire a probe-aware liveness emitter into the `WithHeartbeat` tick: on each tick, when a `*workerprobe.Probe` (or a minimal `RecordEvent`-shaped interface) is available, call `probe.RecordEvent(Event{Kind: &#34;worker.heartbeat&#34;, BeadID: ..., AttemptID: ..., Body: &lt;small JSON&gt;})`. The body should be a compact envelope (worker_id, current_bead, attempt_id, phase, ts) — enough to refresh `last_event_at` server-side and provide minimal context. `RecordEvent` is already non-blocking and best-effort. When no addr / connection refused / no probe, the call must be silent (a single debug-level log line is acceptable) and must not propagate errors to the attempt path.&#xA;&#xA;NON-SCOPE&#xA;- Do not change the probe&#39;s connection lifecycle, register/backfill behavior, or backoff/jitter constants.&#xA;- Do not add a new server endpoint; reuse the existing `/api/workers/&lt;id&gt;/event` path.&#xA;- Do not write the local sidecar (covered by sibling bead ddx-1be8df2b).&#xA;&#xA;Parent: ddx-a3589116. Dependencies: none (independent of ddx-1be8df2b — the two together complete the parent).
