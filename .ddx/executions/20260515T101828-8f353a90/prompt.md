PROBLEM&#xA;Durable DDx audit outputs still appear as uncommitted local dirt after normal worker attempts. Today the live monitor saw `.ddx/metrics/attempts.jsonl` appended and `.ddx/beads.jsonl` outcome rows changed, while the worker continued. The operator-facing behavior must be simple: if DDx writes durable tracked audit state, it commits it; if the state is volatile, it never writes it to tracked files.&#xA;&#xA;ROOT CAUSE WITH FILE:LINE&#xA;- `cli/cmd/execute_loop_shared.go:548-571` appends attempt rows to `.ddx/metrics/attempts.jsonl` through `attemptmetrics.AppendRow` without staging or committing the metrics file.&#xA;- `cli/internal/attemptmetrics/attemptmetrics.go:19` defines `.ddx/metrics/attempts.jsonl` as the durable per-attempt analytics table, so ignoring it would contradict ADR-026.&#xA;- `cli/internal/agent/commit_outcome.go:34-60` handles store mutation errors after outcomes, but successful durable tracker mutations are not wrapped in a transaction that commits the resulting `.ddx/beads.jsonl`, `.ddx/metrics/attempts.jsonl`, and related attachment writes.&#xA;- Existing bead `ddx-051500ff` covers attachment auto-commit; this bead covers the remaining metrics/outcome tracker writes so the system has one coherent tracked-output policy.&#xA;&#xA;PROPOSED FIX&#xA;Introduce a single DDx-managed tracked-output commit path for worker attempts. After an attempt appends attempt metrics or applies a terminal durable tracker mutation, stage only DDx-managed durable paths that changed (`.ddx/beads.jsonl`, `.ddx/beads-archive.jsonl`, `.ddx/metrics/attempts.jsonl`, and attachment files covered by `ddx-051500ff`) and create one audit commit such as `chore: update tracker (execute-bead &lt;attempt-id&gt;)` or a more specific metrics/outcome message. If the commit fails, stop the worker with operator attention instead of continuing to accumulate dirty tracked state.&#xA;&#xA;NON-SCOPE&#xA;- Do not auto-commit arbitrary implementation files.&#xA;- Do not change attempt metrics schema.&#xA;- Do not duplicate attachment reconciliation work owned by `ddx-051500ff`; compose with it or depend on it.&#xA;- Do not hide commit failures behind best-effort warnings.
