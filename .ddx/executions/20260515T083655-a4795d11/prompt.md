    <title>work: fail fast on stale DDx source checkout binary</title>
    <description>
PROBLEM&#xA;Operators can run `ddx work` from the DDx source checkout with an installed binary built from an older commit. That produced live workers still using old readiness/routing behavior even after the local source tree had been fixed. `ddx version` can warn, but `ddx work` and `ddx try` still proceed to claim beads and perform actual builds with stale code.&#xA;&#xA;ROOT CAUSE WITH FILE:LINE&#xA;- `cli/cmd/command_factory.go:603` implements `warnIfInstalledBinaryBehindSource`, but it is only called from the `version` command path at `cli/cmd/command_factory.go:511`.&#xA;- `cli/cmd/command_factory.go:169` runs the global pre-run version gate, but it checks `.ddx/versions.yaml` semver only and does not block a stale installed binary inside the DDx source repository.&#xA;- `cli/cmd/work.go:130` and `cli/cmd/try.go:116` resolve the project root for execution, but neither refuses to build from an older DDx binary when the project root is the DDx source repo.&#xA;&#xA;PROPOSED FIX&#xA;Add an execution preflight for `ddx work` and `ddx try`: when running inside the DDx source repository, and the binary reports a concrete build commit that is an ancestor of the source checkout `HEAD` with one or more commits ahead, fail before queue scan, claim, readiness, or dispatch. The error must name the binary commit, source `HEAD`, project root, and the exact recovery command (`make install` from the source checkout). Keep `ddx version`, `ddx doctor`, `ddx init`, `ddx upgrade`, `help`, and `completion` usable for recovery. Do not make this a warning for worker commands.&#xA;&#xA;IN-SCOPE FILES&#xA;- `cli/cmd/command_factory.go`&#xA;- `cli/cmd/work.go`&#xA;- `cli/cmd/try.go`&#xA;- `cli/cmd/version_gate_test.go`&#xA;- `cli/cmd/work_test.go`&#xA;- `cli/cmd/try_test.go`&#xA;&#xA;NON-SCOPE&#xA;- Do not change release semver gating against `.ddx/versions.yaml`.&#xA;- Do not auto-run `make install` or mutate the operator&#39;s PATH.&#xA;- Do not block dev binaries with `Version=dev` or binaries without a known commit.&#xA;- Do not inspect workers from other projects; this preflight is scoped to the resolved project root.
