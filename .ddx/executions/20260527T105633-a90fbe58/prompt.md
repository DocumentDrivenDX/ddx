    <title>attempt backend: add opt-in in-tree execution mode</title>
    <description>
PROBLEM&#xA;Projects with very large local data, external services, or benchmark fixtures cannot safely rely on the existing DDx attempt backends for every bead. Niflheim is the concrete case: many ready beads are evidence/validation tasks that mostly run tests or long benchmark harnesses against data and services already present in the checked-out tree. The current backends all move execution away from that tree: `worktree`, `local-clone`, and `docker-clone`. That is correct for code-edit isolation, but it makes data-heavy validation fragile or impossible when the attempt needs untracked local datasets, running service state, or a checkout too expensive to duplicate.&#xA;&#xA;ROOT CAUSE WITH FILE:LINE&#xA;- `cli/cmd/work.go:83` and `cli/cmd/try.go:96` expose only `worktree`, `local-clone`, and `docker-clone` as attempt backend choices.&#xA;- `docs/agent-execute.md:97-104` and `docs/helix/01-frame/features/FEAT-010-task-execution.md:776-782` specify the same isolated-backend-only model.&#xA;- There is no documented or implemented backend that runs a single claimed bead directly in the project checkout while still preserving DDx&#39;s claim, evidence, review, commit, and cleanup contracts.&#xA;&#xA;PROPOSED FIX&#xA;Add an explicit opt-in `in-tree` attempt backend for `ddx try` and `ddx work`. It should run the agent in the project checkout itself, not in a linked worktree, local clone, or Docker clone. The backend must be conservative because it trades isolation for access to local data:&#xA;- require a clean git index/worktree before claim, except for DDx-owned tracker/worker metadata that the orchestrator intentionally writes;&#xA;- take a per-project exclusive worker lock so two in-tree attempts cannot run concurrently in the same checkout;&#xA;- record the same attempt evidence and routing/review metadata as isolated backends;&#xA;- after success, stage and commit only files actually changed by the attempt plus the associated tracker/evidence changes;&#xA;- on failure, unclaim/release the bead and leave an explicit rescue message if the tree is dirty rather than silently discarding edits;&#xA;- keep the default backend unchanged (`worktree`) so parallel code-editing remains isolated by default.&#xA;&#xA;NON-SCOPE&#xA;- Replacing the default backend.&#xA;- Making in-tree attempts safe for concurrent workers in one checkout.&#xA;- Solving the existing `docker-clone` remote-daemon bind-mount defect; that is tracked separately by ddx-6967f6e6.&#xA;- Changing Fizeau model routing.&#xA;&#xA;IN-SCOPE FILES&#xA;- `cli/cmd/work.go`&#xA;- `cli/cmd/try.go`&#xA;- `cli/internal/agent/*attempt*` and adjacent tests that select/setup attempt backends&#xA;- `docs/agent-execute.md`&#xA;- `docs/helix/01-frame/features/FEAT-010-task-execution.md`&#xA;&#xA;OUT-OF-SCOPE FILES&#xA;- Fizeau routing code&#xA;- Niflheim benchmark logic&#xA;- Docker runner image contents
    </description>
