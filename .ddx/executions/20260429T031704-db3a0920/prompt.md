    <title>FEAT: agent-logs jsonl schema is metadata-only; add verbose mode capturing tool I/O and content</title>
    <description>
## Problem&#xA;&#xA;`.ddx/agent-logs/agent-claude-*.jsonl` (lean schema) records only:&#xA;  - `session.start` (model, harness, bead_id)&#xA;  - `llm.response` (turn, input_tokens, output_tokens, elapsed_ms)&#xA;  - `tool.call`     (turn, tool name)&#xA;&#xA;Notably absent: model output content, tool call arguments, tool call results, finish_reason, full usage breakdown (cache reads, cache writes), error/stop reasons.&#xA;&#xA;Meanwhile, `agent/demos/sessions/*.jsonl` from the sibling `Projects/agent` repo demonstrates a richer schema for the same event types:&#xA;&#xA;```jsonl&#xA;{&#34;type&#34;:&#34;llm.response&#34;,&#34;data&#34;:{&#34;content&#34;:&#34;...&#34;,&#34;finish_reason&#34;:&#34;tool_calls&#34;,&#34;usage&#34;:{&#34;input&#34;:899,&#34;output&#34;:21,&#34;total&#34;:920},&#34;latency_ms&#34;:3040,&#34;tool_calls&#34;:1}}&#xA;{&#34;type&#34;:&#34;tool.call&#34;,&#34;data&#34;:{&#34;tool&#34;:&#34;edit&#34;,&#34;input&#34;:{&#34;path&#34;:&#34;...&#34;,&#34;old_string&#34;:&#34;...&#34;,&#34;new_string&#34;:&#34;...&#34;},&#34;output&#34;:&#34;Replaced 1 occurrence&#34;,&#34;duration_ms&#34;:0,&#34;error&#34;:&#34;&#34;}}&#xA;```&#xA;&#xA;## Why it matters&#xA;&#xA;- Cannot replay a real session to reproduce a bug — the inputs are gone.&#xA;- Cannot root-cause provider failures (e.g. RotatingKVCache Quantization NYI seen in sessions.jsonl) because the prompt that triggered it is not in the per-event log; only the whole-session metadata in sessions.jsonl has it.&#xA;- The lucebox-hub/dflash `bench_server.py` replay-mode benchmark needs full prompt text from real traces, not just metadata, to be representative.&#xA;- Inability to tell what tool args caused a slow call.&#xA;&#xA;## Proposed fix&#xA;&#xA;Add a verbose mode to the agent-logs writer (env var `DDX_AGENT_LOG_VERBOSE=1` or config flag) that adopts the agent/demos schema:&#xA;  - llm.response: include `content`, `finish_reason`, full `usage` object&#xA;  - tool.call: include `input` (tool args), `output` (truncated to N KB), `duration_ms`, `error`&#xA;  - new event: `llm.request` with messages_count and possibly truncated message preview&#xA;&#xA;Verbose mode off by default to avoid bloating logs. On for any session with `--debug` or for failed sessions (auto-bump-on-error).&#xA;&#xA;## Side note&#xA;&#xA;`sessions.jsonl` (the 37 MB whole-session log) does have full `prompt` text but no per-turn breakdown. So we have two log formats and neither captures the full picture. This bead is about the per-event log only; whole-session log is out of scope.
    </description>
    <acceptance>
1. `DDX_AGENT_LOG_VERBOSE=1 ddx agent ...` produces a trace where llm.response events include `content`, `finish_reason`, `usage`, and tool.call events include `input`, `output`, `duration_ms`, `error`.&#xA;2. Schema documented in cli/internal/agent/session_log_format.go (Go doc comment listing all fields per event type for both lean and verbose modes).&#xA;3. Default behavior unchanged (lean mode by default; existing tooling that consumes the lean schema continues to work).&#xA;4. Test fixture added for verbose mode that asserts every required field is populated.
    </acceptance>
