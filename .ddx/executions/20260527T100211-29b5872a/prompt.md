    <title>fix executor: enforce parent-death cleanup for execute-bead harness children on Linux</title>
    <description>
PROBLEM&#xA;When a `ddx work` orchestrator dies abnormally (SIGKILL, crash, OOM, or supervising-process loss) while an execute-bead attempt is running, the spawned harness child can reparent to PID 1 and continue indefinitely. Observed 2026-05-27: a `claude --print` execute-bead child remained orphaned with PPID 1 for ~9h53m after its `ddx work` parent exited.&#xA;&#xA;ROOT CAUSE (file:line)&#xA;- cli/internal/agent/executor.go:156 spawns the harness via `exec.Command(binary, args...)` and then calls `cmdSetProcessGroup(cmd)`.&#xA;- cli/internal/agent/executor_unix.go:13-15 sets only `SysProcAttr{Setpgid: true}`. There is no Linux `Pdeathsig`, so the kernel does not signal the harness when the parent dies abnormally.&#xA;- cli/internal/agent/executor_unix.go:20-25 kills the process group only from graceful executor paths.&#xA;- cli/cmd/root.go:49-54 uses `signal.NotifyContext` for os.Interrupt and SIGTERM only. SIGKILL and crashes cannot be caught, so executor cleanup does not run.&#xA;&#xA;PROPOSED FIX&#xA;On Linux, set `SysProcAttr.Pdeathsig` to SIGKILL or SIGTERM for execute-bead harness commands in `cmdSetProcessGroup`. Handle the Go runtime thread-migration caveat with `runtime.LockOSThread` around process start or an equivalent parent-PID guard so the parent-death signal is attached to the intended parent thread. Preserve process-group setup so existing graceful cancellation and timeout cleanup still kill the whole harness tree. Document the macOS/Windows fallback because `Pdeathsig` is Linux-only.&#xA;&#xA;NON-SCOPE&#xA;- Graceful SIGINT/SIGTERM shutdown already works through executor.go:231-255; do not regress it.&#xA;- The harness-runner-internal leak in fizeau is covered by fizeau-50b6f055; this child bead is only the DDx orchestrator-to-harness-child boundary.&#xA;- Startup orphan reaping is covered by a separate child bead from this decomposition.&#xA;- docker-clone backend cleanup is covered by a separate child bead from this decomposition.&#xA;&#xA;CROSS-REF&#xA;- fizeau-50b6f055: fizeau harness-runner process-group cleanup.&#xA;- ddx-8f2e0ebf: lease release / operator-attention.
