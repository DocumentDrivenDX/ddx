Open decisions from the epic description, resolved here so children are
implementable without re-litigation:
- Archival trigger: file-size threshold, default >4MB, on closed beads only;
  --max-size / --older-than / --max-count flags allow overrides.
- Closed-bead events: move to attachment sidecars on archive (keeps archive
  rows small, which is the dominant size driver per the epic description).
- Migration semantics: explicit `ddx bead migrate-archive` command sharing
  logic with `ddx bead archive`; idempotent + dry-run.

The parent epic now depends on all six children; it will close cleanly once
they do. AC item 1 ("all child beads closed") is therefore the literal
remaining gate for the parent.
