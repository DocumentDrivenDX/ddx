Relevant code paths examined in this worktree:

1. cli/cmd/agent_execute_loop_escalation.go:299 — runEscalatingPowerAttempts short-circuits on report.Disrupted=true and never retries at the same power. provider_connectivity sets Disrupted=true (cli/internal/agent/execute_bead_loop.go:2715-2718), so the in-execution path never attempts the alternate at the same power class.

3. cli/internal/agent/execute_bead_loop.go:4733 — CheckAndApplyRouteExclusions (called on the NEXT claim, cli/cmd/execute_loop_shared.go:449) builds ExcludedRoutes from the failed-route list and re-runs ResolveRoute at the same minPower. In theory this should pick up the alternate on the next iteration; in the user's repro, it did not. Root cause for that gap is not visible from static read of the code (could be Available/AutoRoutable catalog state, worker iteration ordering, or a pausedInfraUntil interaction) and requires reproduction before fixing.
