    <title>ddx work / agent execute-loop: stay alive while queue has ready work and viable providers exist</title>
    <description>
Workers exit prematurely even when ready beads remain in the queue. Operators have to manually relaunch, defeating the point of a long-running drain.&#xA;&#xA;## Root causes (from sub-agent investigation)&#xA;&#xA;1. **cli/internal/agent/execute_bead_loop.go:440-441**: when nextCandidate returns no eligible bead AND runtime.PollInterval &lt;= 0, the loop returns immediately. Combined with the per-Run &#39;attempted&#39; map (line 388, 476), once each currently-ready bead has been *attempted once* the loop exits even though those same beads (or new arrivals) are still queue-ready.&#xA;&#xA;2. **cli/internal/agent/execute_bead_loop.go:483-508**: routing-preflight rejection on a single bead exits the entire loop (&#39;return result, nil&#39;), abandoning all other ready beads.&#xA;&#xA;3. **Default --poll-interval=0 everywhere**: cli/cmd/work.go:66, cli/cmd/agent_cmd.go:1568, cli/internal/server/graphql_adapters.go:115, cli/internal/server/server.go:2281. All three recent server-spawned workers in .ddx/workers/ show poll_interval: &#39;0s&#39;, so cause #1 fires the first empty poll.&#xA;&#xA;## Fix shape (locked decisions baked in)&#xA;&#xA;- Flip default --poll-interval from 0 → **30s** everywhere (CLI + server adapter). Operator can override with --once for one-shot or explicit --poll-interval=N.&#xA;- Reset per-Run &#39;attempted&#39;/&#39;hookFailed&#39; maps when the loop falls through to &#39;no candidate&#39;, before sleeping. Prevents the &#39;attempted-once = exit&#39; trap.&#xA;- Convert preflight early-exit (483-508) to per-bead &#39;continue&#39; so one bead&#39;s routing failure doesn&#39;t abandon the rest.&#xA;- Add structured exit_reason to loop.end event; reserve &#39;providers_exhausted&#39; slot for ddx-aede917d (quota-pause work).&#xA;- Server-managed workers expose &#39;no_ready_work&#39; as a **transient &#39;running (idle)&#39; substate**, not terminal. Worker stays alive; UI shows running (idle).&#xA;- Preserve --once and explicit --poll-interval=0 for back-compat (legacy &#39;drain-and-exit&#39; semantics).&#xA;&#xA;Acceptable exit conditions remain (per existing beads):&#xA;- SIGTERM / SIGINT (operator-driven)&#xA;- All providers durably out (per ddx-aede917d quota pause/resume — workers pause+probe, not exit)&#xA;- No providers configured at all (fatal config error)&#xA;- --once explicitly requested&#xA;- --poll-interval=0 explicitly requested
    </description>
