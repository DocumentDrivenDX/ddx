For Gemini installed through the Node wrapper, argv[0] is `node` and argv[1] is the `gemini` CLI path, so `providerForCommand` returns empty. Existing tests at `cli/internal/agent/provider_children_test.go:198-240` cover direct executable names (`gemini`) but not `node .../gemini`, and the process-group test at `cli/internal/agent/provider_children_test.go:242-292` uses argv[0]=gemini, so it also misses the real wrapper shape.&#xA;&#xA;PROPOSED FIX&#xA;Teach provider child classification to recognize known provider CLI paths passed as wrapper script arguments, starting with Node-wrapped Gemini (`node /path/to/gemini ...`).
