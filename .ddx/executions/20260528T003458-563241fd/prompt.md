    <title>cleanup: reap expired run-state for dead worker PIDs</title>
    <description>
PROBLEM&#xA;`ddx cleanup --apply` can leave an expired per-attempt run-state file behind and continue treating the corresponding attempt worktree as live even after the worker process is gone. This keeps the project checkout dirty (`?? .ddx/run-state/`) and prevents reliable follow-up workers until an operator manually deletes the stale file.&#xA;&#xA;OBSERVED EVIDENCE&#xA;In `/home/erik/Projects/niflheim`, a canceled `ddx work --once` left `.ddx/run-state/20260521T071245-feb2ad20.json` with `pid: 4109595`, `refreshed_at: 2026-05-21T07:12:48Z`, and `expires_at: 2026-05-21T07:14:48Z`. After the process was gone and the expiry time had passed, `ddx cleanup --apply` reported `removed_run_state_files: 0` and still classified `/home/erik/.cache/ddx/exec-wt/.execute-bead-wt-niflheim-439a49ae-20260521T071245-feb2ad20` as `preserved_live_attempt` with message `matched live run-state`.&#xA;&#xA;ROOT CAUSE WITH FILE:LINE&#xA;`cli/cmd/execute_loop_shared.go:616-623` only treats run-state freshness/expiry as the liveness predicate for cleanup preservation. It does not require the recorded `RunState.PID` to still be alive when deciding that an expired/dead attempt should be preserved. `cli/cmd/cleanup_test.go:77-180` covers run-state cleanup in simpler cases but does not reproduce an expired per-attempt run-state whose PID is dead while the worktree still matches that run-state.&#xA;&#xA;PROPOSED FIX&#xA;Make cleanup treat a per-attempt run-state as live only when both conditions hold: the lease is fresh/non-expired and the recorded worker PID is still alive. Expired run-state files for dead PIDs must be removed, and matching attempt worktrees must be eligible for stale cleanup. Keep live active attempts protected.&#xA;&#xA;NON-SCOPE&#xA;- Changing work status enrichment for live workers.&#xA;- Changing worker heartbeat cadence.&#xA;- Deleting completed evidence bundles.&#xA;- Reworking global temp-root cleanup across projects.&#xA;&#xA;IN-SCOPE FILES&#xA;- `cli/cmd/execute_loop_shared.go`&#xA;- `cli/cmd/cleanup_test.go`&#xA;- Adjacent cleanup/startup-housekeeping tests if needed.
