Workaround note: `ddx bead create` from inside an execute-bead worktree
mis-derived the bead ID prefix from the worktree directory basename
(e.g. ".execute-bead-wt-ddx-5d49b14e-...-<hex>") because
`git rev-parse --show-toplevel` from a linked worktree returns the worktree
path, not the project root. Setting DDX_BEAD_PREFIX=ddx as an env override
produced correct ddx-* IDs. Four malformed-ID beads created during the
diagnosis pass were closed. detectPrefix() in cli/internal/bead/store.go
already has the intent of running git from the project root, but the
caller-provided workingDir in this invocation path resolves to the worktree;
worth a follow-up bead, but out of scope for this decomposition attempt.
