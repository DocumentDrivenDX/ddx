    <title>bead: add atomic Store.Release so failure-release clears the claim and status reflects the lease</title>
    <description>
PROBLEM&#xA;When a worker abandons a bead (route-resolution timeout, watchdog fire, or any failure release), the bead&#39;s claim/lease must be cleared atomically and `ddx bead status` must reflect the lease state. Claiming a bead today atomically transitions it to in_progress, but there is no symmetric release path that, in a single locked write, transitions the bead back and clears the lease. The parent bug (ddx-8f2e0ebf) observed &#34;bead has a claim but ddx bead status shows In progress: 0&#34; — claim and lifecycle status not atomically coupled on the release side.&#xA;&#xA;ROOT CAUSE&#xA;cli/internal/bead/store.go: Store.Claim (~lines 863-930) is atomic — under WithLock it calls transitionLifecycleInPlace(StatusInProgress) + sets Owner + WriteAll, then writeClaimHeartbeat writes the lease sidecar. There is NO symmetric Store.Release that, under the same WithLock, transitions status back (default open), clears Owner, WriteAll, AND removes the claim-heartbeat lease sidecar in one critical section. Failure-release paths can leave a stale lease sidecar while status reports not-in_progress, producing the phantom-lease inconsistency.&#xA;&#xA;PROPOSED FIX&#xA;1. Add Store.Release(id, assignee, targetStatus) mirroring Store.Claim: under WithLock, read beads, transitionLifecycleInPlace to target status (open by default), clear Owner, WriteAll, and remove the claim-heartbeat lease sidecar — all in one locked operation.&#xA;2. Route the worker&#39;s failure-release paths through Store.Release so claim clearing and status are coupled.&#xA;3. Ensure `ddx bead status` counts reflect lease state: a released bead is not counted in_progress and has no owner; a claimed bead is in_progress with an owner.&#xA;&#xA;NON-SCOPE&#xA;Route-resolution timeout (sibling child A), heartbeat watchdog (child B), same-bead-twice guard (child D), operator-attention query surface + docs (child E). This bead only establishes atomic claim/release symmetry and status accuracy. This is a decomposition child of ddx-8f2e0ebf.
    </description>
