    <title>work: keep live worker state out of tracked DDx files</title>
    <description>
PROBLEM&#xA;`ddx work --watch` currently dirties tracked files while a worker is merely alive. Operators then see `.ddx/beads.jsonl` and `.ddx/run-state*` changes even when no durable bead outcome has happened, and concurrent watchers can race the same moving state.&#xA;&#xA;ROOT CAUSE WITH FILE:LINE&#xA;- `cli/internal/bead/store.go:869-925` `ClaimWithOptions` persists live claim state (`status=in_progress`, `owner`, `claimed-at`, `claimed-pid`, `claimed-machine`, `claimed-session`, `claimed-worktree`, `work-heartbeat-at`) into `.ddx/beads.jsonl` before touching the external lease.&#xA;- `cli/internal/bead/claim_liveness.go:15-24` and `cli/internal/bead/claim_liveness.go:64-75` already write an external lease, but the lease only carries `bead_id`, `updated_at`, and `pid`, so operator-facing claim metadata still depends on tracker mutation.&#xA;- `cli/internal/agent/run_state.go:100-125` writes live attempt state under `.ddx/run-state/` and `.ddx/run-state.json`; `cli/cmd/init.go:18-31` ignores these paths for new repos, but existing tracked run-state files still produce dirty deletions when cleanup runs.&#xA;- `cli/internal/agent/worker_liveness_sidecar_test.go:22-28` currently documents the tracker as the durable claim marker; that contract now conflicts with the desired live-state policy.&#xA;&#xA;PROPOSED FIX&#xA;Move worker-owned live claim state fully into sidecar state. Extend the claim lease record to carry owner/session/worktree/machine/started_at/pid/updated_at and make the `ddx work` / `ddx try` worker claim path update only that lease until a terminal durable outcome occurs. `ReadyExecution` must exclude beads with a fresh sidecar lease and include beads whose sidecar lease is absent or stale. Manual tracker lifecycle operations can remain explicit tracker operations, but ordinary worker liveness must not dirty `.ddx/beads.jsonl`.&#xA;&#xA;Also migrate run-state policy so live run-state files are never tracked: add a migration/cleanup path that removes tracked `.ddx/run-state.json` and `.ddx/run-state/*.json` from the git index, keeps the ignore rules, and proves cleanup does not create tracked deletions.&#xA;&#xA;NON-SCOPE&#xA;- Do not change bead close/proposed/blocked/cancelled semantics.&#xA;- Do not drop execution evidence or attempt metrics.&#xA;- Do not remove operator visibility; surface live owner/bead/attempt from sidecars instead of the tracker.&#xA;- Do not manually edit `.ddx/beads.jsonl` in tests or implementation.
